Lesson 21 · Gameplay · Lives & respawn

Three lives, a respawn, and a moment of mercy

Right now one depleted health bar ends the run. Real Space Invaders gives you lives: lose your health, respawn, try again — with a brief flicker of invulnerability so you're not instantly shot again. The good news: you built the foundation in L20. takeDamage, die, isAlive already exist. We're growing Player from "has health" into a small, self-contained state machine — and keeping every rule behind its interface.

The model: two layers of "alive"

The key design idea is distinguishing two questions that isAlive() currently conflates:

QuestionMeansDriven by
isAlive()is this life still going?hp > 0
isGameOver()is the run over?lives <= 0

Losing all hp no longer ends the game — it spends a life and respawns. The run ends only when lives run out. The scene will transition on isGameOver(), not !isAlive().

Step 1 — grow the Player's private state

Three new private members; all the existing public methods stay, plus two queries and a private helper. Player owns the respawn position now (it needs it to put itself back):

// game.h
class Player {
public:
  Vector2 pos{};
  Vector2 dir{};
  float speed{};
  float radius{};

  void reset(Vector2 startPosition);     // new game: full lives + spawn
  void update(float deltaTime);
  void draw(const Texture2D& texture) const;
  void takeDamage(int amount = 1);
  void die();

  bool isAlive() const { return hp > 0; }
  bool isVulnerable() const { return invulnerableTimer <= 0.0f; }   // NEW
  bool isGameOver() const { return lives <= 0; }                    // NEW
  int  health() const { return hp; }
  int  livesRemaining() const { return lives; }                    // NEW (HUD)

private:
  void respawn();                        // NEW: internal — not part of the public interface

  int hp{};
  int lives{};                           // NEW
  float invulnerableTimer{};             // NEW: seconds of mercy remaining
  Vector2 startPosition{};               // NEW: where to respawn
};
Public vs private interface. respawn() is private on purpose. It's a mechanism the player triggers internally (after losing a life) — nothing outside should be able to say "respawn now." The public surface stays small and intention-revealing (takeDamage, die, the queries); the moving parts hide behind it. This is the L16 lesson applied to behaviour, not just data.

Step 2 — the life-cycle methods

Add the constants by the other gameplay ones, then wire the state transitions. Notice reset and respawn share the spawn logic, so reset stores the position and delegates:

constexpr int PLAYER_LIVES = 3;
constexpr float INVULNERABILITY_TIME = 2.0f;
// scene_gameplay.cpp
void Player::reset(Vector2 start) {
  startPosition = start;
  speed = PLAYER_SPEED;
  radius = PLAYER_RADIUS;
  lives = PLAYER_LIVES;
  respawn();                       // position + hp + mercy window
}

void Player::respawn() {
  pos = startPosition;
  dir = {0.0f, 0.0f};
  hp = PLAYER_HEALTH;
  invulnerableTimer = INVULNERABILITY_TIME;
}

void Player::takeDamage(int amount) {
  if (!isVulnerable()) return;     // mercy: ignore hits right after respawn
  hp -= amount;
  if (hp <= 0) {
    hp = 0;
    loseLife();
  }
}

void Player::die() {               // breach / body-block: skip hp, spend a life now
  hp = 0;
  loseLife();
}

void Player::loseLife() {
  --lives;
  if (lives > 0) respawn();        // still have lives -> back in; else stay dead (isGameOver)
}

(Add void loseLife(); to the private section too.) Trace it: a bullet hit while vulnerable drops hp; when hp hits 0, loseLife decrements lives and either respawns (lives left) or leaves the player dead with lives == 0 → isGameOver() true.

Step 3 — tick the mercy timer

The invulnerability window counts down in update — the same accumulator pattern as the enemy fire cooldown. One line at the end of Player::update:

void Player::update(float dt) {
  // ...existing movement code unchanged...
  if (invulnerableTimer > 0.0f) invulnerableTimer -= dt;
}

Step 4 — show the mercy: blink while invulnerable

A respawned player should visibly flicker so the player understands they're briefly safe. Skip drawing on alternating slices of time using fmodf (floating-point modulo, from <cmath>):

// scene_gameplay.cpp — add #include <cmath> at the top
void Player::draw(const Texture2D& texture) const {
  // blink: while invulnerable, hide for every other 0.1s slice
  if (!isVulnerable() && fmodf(invulnerableTimer, 0.2f) > 0.1f) return;

  drawOffset(texture, pos, WHITE);
  DrawCircleLinesV(pos, radius, RED);
}

(!isVulnerable() means "currently in the mercy window"; the fmodf term toggles roughly five times a second. When the timer expires, isVulnerable() is true and the guard never fires, so the player draws solid again.)

Step 5 — the scene transitions on game-over, and the HUD shows lives

Swap the end-of-life check for the end-of-run check, and pass lives to the HUD:

// SceneGameplay::update — the transition condition
if (state.victory || state.player.isGameOver())      // was: !state.player.isAlive()
  return std::make_unique<SceneGameover>();

// SceneGameplay::draw
drawHud(state.player.health(), state.player.livesRemaining(), state.score);

Extend the HUD (a free function over primitives — still no game types, per L11):

// hud.h
void drawHud(int health, int lives, int score);

// hud.cpp — add a lives readout
DrawText(TextFormat("Lives: %i", lives), HUD_MARGIN, GetScreenHeight() - HUD_MARGIN * 4,
         HUD_FONT_SIZE, HUD_TEXT_COLOR);

Files you'll touch (checklist)

  1. include/game.h — add isVulnerable, isGameOver, livesRemaining queries; private respawn(), loseLife(), lives, invulnerableTimer, startPosition.
  2. src/scene_gameplay.cpp — #include <cmath>; constants PLAYER_LIVES/INVULNERABILITY_TIME; rewrite reset, add respawn/loseLife, update takeDamage/die; tick the timer in update; blink in draw; transition on isGameOver(); pass lives to drawHud.
  3. include/hud.h + src/hud.cpp — add the lives parameter + a readout.
  4. Build & play: lose all 5 hp → flicker + respawn at start, two more times; HUD shows 3→2→1; the run ends only when lives hit 0; you can't be killed during the flicker.
Traps: (1) Store startPosition in reset before calling respawn — otherwise the first respawn sends the player to {0,0}. (2) die() deliberately bypasses the isVulnerable check (a breach should always cost a life) — but routes through loseLife so it can't skip the lives accounting. (3) Gate the transition on isGameOver(), not !isAlive() — the latter is now true on every respawn frame and would end the game instantly. (4) fmodf needs <cmath>; plain fmod is the double version (works, but match the float).
