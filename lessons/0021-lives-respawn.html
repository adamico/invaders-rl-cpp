Lesson 21 · Gameplay · Lives & respawn

Three lives, a respawn, and a moment of mercy

Classic Space Invaders has no health bar: one hit and you lose a life, respawn, try again — with a brief flicker of invulnerability so you're not instantly shot again. You spent L20 building the foundation (takeDamage, die already exist), and L21's prep gave Player its own translation unit. Now we replace the old hp counter with the authentic model: lives, and the two distinct ways to lose them.

Design decision (you made this call). The old code carried hp = 5 as a tiny health bar — non-canonical for Invaders, and stacking lives on top of it meant two nested resources (5 hits × 3 lives = 15). We're going classic: drop hp entirely. One resource — lives. One hit costs a life. Cleaner, authentic, and it removes the redundancy you spotted.

The model: two ways to lose, one resource

With hp gone, lives is the only health state. Two distinct events drain it, and giving them different methods makes the intent obvious:

EventMethodEffect
any physical hit — enemy bullet, or touching an enemytakeDamage()lose one life, respawn if any remain
swarm reaches the bottom (breach)die()instant game over — you're overrun, no respawn

The player reports isDead() (lives <= 0) about itself; the scene decides that a dead player means game over. "Game over" is the scene's vocabulary, not the player's — the entity exposes state, the higher layer interprets it (model vs view, as with feedback/sound earlier).

Step 1 — replace hp with lives

(Player now lives in player.h/player.cpp.) Remove hp, health(), and isAlive(); add the lives state, the queries, and two private helpers:

// player.h  — replace PLAYER_HEALTH with these, beside the other PLAYER_* constants
constexpr int PLAYER_LIVES = 3;
constexpr float INVULNERABILITY_TIME = 2.0f;

class Player {
public:
  Vector2 pos{};
  Vector2 dir{};
  float speed{};
  float radius{};

  void reset();                          // new game: full lives + spawn (no-arg, self-places)
  void update(float deltaTime);
  void draw(const Texture2D& texture) const;
  void takeDamage();                     // CHANGED: no int arg — one hit = one life
  void die();

  bool isVulnerable() const { return invulnerableTimer <= 0.0f; }   // NEW
  bool isDead() const { return lives <= 0; }                        // NEW — about the PLAYER, not the game
  int  livesRemaining() const { return lives; }                    // NEW (HUD)

private:
  void respawn();                        // NEW: internal — not the public interface
  void loseLife();                       // NEW: internal

  int lives{};                           // the only resource now
  float invulnerableTimer{};             // seconds of mercy remaining
};
Public vs private interface. respawn() and loseLife() are private on purpose — internal mechanisms, triggered by the player's own logic. Nothing outside should say "respawn now" or "decrement a life directly." The public surface stays intentional (takeDamage, die, queries); the moving parts hide behind it. L16's encapsulation lesson, applied to behaviour.

Step 2 — the life-cycle methods

reset sets the run-level state (lives) and delegates per-life setup to respawn, which self-places exactly like the old reset did. The two drain paths route through loseLife so the accounting lives in one place:

// player.cpp
void Player::reset() {
  speed = PLAYER_SPEED;
  radius = PLAYER_RADIUS;
  lives = PLAYER_LIVES;
  respawn();                       // position + mercy window
}

void Player::respawn() {
  pos = {GetScreenWidth() / 2.0f, GetScreenHeight() - (PLAYER_RADIUS * 4)};  // self-place
  dir = {0.0f, 0.0f};
  invulnerableTimer = INVULNERABILITY_TIME;
}

void Player::takeDamage() {
  if (!isVulnerable()) return;     // mercy: ignore hits right after respawn
  loseLife();
}

void Player::die() {               // breach only: overrun, instant game over, no respawn
  lives = 0;
}

void Player::loseLife() {
  --lives;
  if (lives > 0) respawn();        // lives left -> back in; else stay dead (isGameOver)
}

Trace it: a bullet hit while vulnerable calls takeDamage → loseLife → decrement, then respawn (lives remain) or stay dead (lives == 0 → isDead() true). A breach calls die() → straight to lives = 0, instant game over. Touching an enemy goes through takeDamage() like any other hit — only the breach is unsurvivable.

Step 3 — tick the mercy timer

The invulnerability window counts down in update — same accumulator pattern as the enemy fire cooldown. One line at the end of Player::update (in player.cpp):

void Player::update(float dt) {
  // ...existing movement code unchanged...
  if (invulnerableTimer > 0.0f) invulnerableTimer -= dt;
}

Step 4 — show the mercy: blink while invulnerable

A respawned player should visibly flicker so the safety is legible. Skip drawing on alternating slices of time using fmodf (float modulo, from <cmath>):

// player.cpp — add #include <cmath> at the top
void Player::draw(const Texture2D& texture) const {
  // blink: while invulnerable, hide for every other 0.1s slice
  if (!isVulnerable() && fmodf(invulnerableTimer, 0.2f) > 0.1f) return;

  Vector2 drawPos = Vector2Add(pos, CANVAS_OFFSET);
  DrawTextureV(texture, drawPos, WHITE);
  DrawCircleLinesV(pos, radius, RED);
}

(!isVulnerable() means "currently in the mercy window"; the fmodf term toggles about five times a second. When the timer expires, isVulnerable() is true and the guard never fires — the player draws solid again.)

Step 5 — scene transitions on game-over; HUD shows lives not health

Swap the end-of-life check for end-of-run, and replace the health readout with lives:

// SceneGameplay::update — the SCENE interprets a dead player as game over
if (state.victory || state.player.isDead())          // was: !state.player.isAlive()
  return std::make_unique<SceneGameover>();

// SceneGameplay::draw
drawHud(state.player.livesRemaining(), state.score); // was: state.player.health()

And the HUD (a free function over primitives — still no game types, per L11): swap the health parameter for lives.

// hud.h
void drawHud(int lives, int score);

// hud.cpp — "Lives: N" instead of "Health: N"
DrawText(TextFormat("Lives: %i", lives), HUD_MARGIN, GetScreenHeight() - HUD_MARGIN * 2,
         HUD_FONT_SIZE, HUD_TEXT_COLOR);

Files you'll touch (checklist)

  1. include/player.h — drop PLAYER_HEALTH; add PLAYER_LIVES/INVULNERABILITY_TIME; remove hp/health()/isAlive(); add takeDamage() (no arg), isVulnerable, isDead, livesRemaining, private respawn()/loseLife()/lives/invulnerableTimer.
  2. src/player.cpp — #include <cmath>; rewrite reset (delegates to respawn), add respawn/loseLife, rewrite takeDamage/die; tick timer in update; blink in draw.
  3. src/scene_gameplay.cpp — transition on player.isDead(); drawHud(livesRemaining(), score). Both hit paths call takeDamage() (now arg-free): the enemy-bullet collision AND the player-enemy body contact (change that site's die() → takeDamage()). Only the breach check calls die().
  4. include/hud.h + src/hud.cpp — parameter health → lives, label "Lives".
  5. Build & play: one bullet = one life lost + flicker + respawn; HUD shows 3→2→1; a breach or touching an enemy ends the run instantly; you can't be killed during the flicker.
Traps: (1) die() deliberately bypasses the isVulnerable check — being overrun should always end the run, mercy window or not. (2) Gate the transition on player.isDead() (the scene names "game over"; the player only reports its own death) — and make sure every old isAlive()/health() caller is updated, or it won't compile (that's the point — the compiler finds them for you). (3) takeDamage lost its int parameter; update the call site. (4) fmodf needs <cmath> (plain fmod is the double version).
