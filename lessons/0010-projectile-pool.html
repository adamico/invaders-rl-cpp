What an object pool is

Your bullets are already a pool, you just never named it. A pool is a fixed-size block of pre-allocated objects, each with an active flag; instead of allocating/freeing at runtime, you flip flags. std::array<Projectile, MAX_PROJECTILES> + the active bool + "find the first inactive slot to spawn" is the textbook pattern. It avoids per-shot heap allocation and keeps the bullets contiguous for cache-friendly iteration. [Game Programming Patterns — Object Pool]

Today's three loose bullet functions (InitBullets, UpdateProjectiles, DrawProjectiles) plus the slot-finding loop in PlayerShoot are exactly the same "invariant cluster" smell as the swarm. Same fix: one class.

The class — mirror of Swarm

// NEW FILE: include/projectile_pool.h
#pragma once
#include "game.h"   // for Projectile, MAX_PROJECTILES

struct ProjectilePool {
  std::array<Projectile, MAX_PROJECTILES> bullets{};

  void reset();                          // was InitBullets
  bool fire(Vector2 from);                 // was the slot loop in PlayerShoot — returns "did it fire?"
  void update(float dt);                 // was UpdateProjectiles
  void draw(const Texture2D& texture) const;  // was DrawProjectiles
};

Why fire returns a bool — same decoupling as Swarm::update

The old PlayerShoot did two unrelated things: it played the laser sound, and it found a bullet slot. Sound is an audio/scene concern; the pool only knows about slots. So the pool's fire reports whether a slot was free (true = a bullet launched), and the caller decides whether to play the sound. The pool never touches GameResources or audio. Identical "report a fact, let the caller act" pattern you used for the swarm's breach. As a bonus you fix a latent bug: the old code played the sound even when the pool was full and no bullet actually fired.

The bodies (relocated, logic unchanged)

// NEW FILE: src/projectile_pool.cpp
#include "projectile_pool.h"

constexpr float PROJECTILE_SPEED = 500.0f;   // move these out of scene_gameplay.cpp
constexpr float PROJECTILE_RADIUS = 5.0f;

void ProjectilePool::reset() {
  for (Projectile& b : bullets) b = { .active = false };
}

bool ProjectilePool::fire(Vector2 from) {
  for (Projectile& b : bullets) {
    if (b.active) continue;
    b.spawn(from);          // Projectile::spawn already sets speed/radius/dir (lesson 5)
    return true;            // launched
  }
  return false;             // pool full — nothing fired
}

void ProjectilePool::update(float dt) {
  for (Projectile& b : bullets) b.update(dt);
}

void ProjectilePool::draw(const Texture2D& texture) const {
  for (const Projectile& b : bullets) b.draw(texture);
}

Note PROJECTILE_SPEED/RADIUS are used by Projectile::spawn, which already lives in scene_gameplay.cpp today. Keep spawn's definition wherever it is and make sure the constants are visible to it — simplest is to leave Projectile::spawn in scene_gameplay.cpp with its constants, and have fire just call it. If you'd rather move Projectile::spawn into projectile_pool.cpp too, move the constants with it. Either works; don't duplicate the constants in two files.

Rewire scene_gameplay.cpp

Delete InitBullets, UpdateProjectiles, DrawProjectiles. Then:

// InitGameplay
state->projectiles.reset();   // was InitBullets(state);

// PlayerShoot — caller owns the sound now
void PlayerShoot(GameState& state) {
  if (!IsKeyPressed(KEY_SPACE)) return;
  if (state.projectiles.fire(state.player.pos))      // only if a bullet actually launched
    PlaySound(state.resources.laserSound);
}

Collisions stay free — you already know why

CheckBulletEnemyCollisions straddles the pool and the swarm. It belongs to neither class; it stays a non-member that reaches into both state.projectiles.bullets and state.swarm.enemies. Same C.4/C.5 judgment from lesson 9 — nothing new, just confirm you'd reach the same conclusion unprompted.

Files you'll touch (checklist)

  1. include/projectile_pool.h (new) — the ProjectilePool struct.
  2. src/projectile_pool.cpp (new) — reset/fire/update/draw bodies + projectile constants.
  3. include/game_state.h — include the new header; swap the bullets array for ProjectilePool projectiles;.
  4. src/scene_gameplay.cpp — delete the three bullet free-functions; rewire InitGameplay, PlayerShoot (sound on fire()==true), update, draw, and the collision loop (state.projectiles.bullets).
  5. No Makefile change — src/*.cpp globbed.
  6. Build & play: hold space (bullets cap at MAX_PROJECTILES, sound only when one actually fires), kill enemies, full cycle.
Traps: (1) grep for leftover state.bullets → all become state.projectiles.bullets. (2) Duplicated PROJECTILE_* constants in two files → keep one home. (3) Forgetting projectiles.reset() in InitGameplay → on restart, old bullets may still be flagged active.
