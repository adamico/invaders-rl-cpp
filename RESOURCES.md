# Resources

## Primary
- **C++ Core Guidelines — Resource Management (R.1, R.5, C.31)** — https://isocpp.github.io/CppCoreGuidelines/CppCoreGuidelines#S-resource
  The canonical "why RAII" reference. R.1: manage resources automatically using RAII.
- **cppreference — RAII** — https://en.cppreference.com/w/cpp/language/raii
  Definition, rules, and the rule of three/five it implies.

## Supporting
- **raylib cheatsheet** — https://www.raylib.com/cheatsheet/cheatsheet.html
  Authoritative list of LoadTexture/UnloadTexture, LoadSound/UnloadSound signatures.
- **cppreference — rule of three/five/zero** — https://en.cppreference.com/w/cpp/language/rule_of_three
  Why an RAII class that owns handles must control (or delete) copy/move.
